About Attributes

Today we will look at the some attributes in HTML. Attributes provide additional information about HTML elements and are always specified in the start tag. They come in name/value pairs like this: name="value".

Attribute: id is used to uniquely identify an element in the document. It can be used for styling with CSS or for scripting with JavaScript. Important! The value of the id attribute must be unique within the document. For example, <div id="uniqueElement">Content</div>. Important! Attribute: id name can't be a number. Also, it can't start with number. but, it can contain numbers in the middle or at the end of the name. For example, id="element1" is valid, but id="1element" is not valid.

We all know that the class attribute is used to assign one or more class names to an element. These class names can be used by CSS and JavaScript to apply styles or manipulate the element. For example, <span class="highlight">Highlighted Text</span>. Unlike attribute id, the value of the class attribute does not have to be unique. Multiple elements can share the same class name, allowing for bulk styling or scripting.

Sometimes, we need to use attribute title to provide additional information about an element. This information is typically displayed as a tooltip when the user hovers over the element. For example, <a href="#" title="This is a link">Link</a>. But, in nowadays in web development, the title attribute is often used less frequently, as other methods such as ARIA attributes or JavaScript tooltips provide more flexibility and control over the user experience.

Sometimes we need to use attribute: style to apply inline CSS styles directly to an element. This allows for quick styling without the need for external or internal stylesheets. For example, <div style="color: red; font-size: 20px;">Styled Text</div>. However, it's generally recommended to use external stylesheets for better maintainability and separation of concerns. Important! The style attribute should not be used for large-scale styling, as it can lead to code that is difficult to read and maintain. For that, we should use external CSS files or internal stylesheets within the <style> tag in the document's head section.
