About Tags and Attributes in HTML

With time in HTML appear new ones tags and attributes, which are used to create more complex and functional web pages. But, not all new tags and attributes are supported by all browsers, so it is important to use them correctly and check their compatibility.

Some tags and attributes with time become deprecated, which means that they are no longer recommended for use and may not work in future versions of HTML. It is important to keep up with the latest standards and use only those tags and attributes that are supported by modern browsers. And, if we use them, our page users can't see the content correctly. And, we lose page stability and functionality.

we may encounter a situation where some web browsers do not support certain tags and attributes. In such cases, it is important to provide fallback solutions or alternative content to ensure a consistent user experience across different browsers.

If, we want to use some tag, but, we don't know if it is supported by all browsers, we can use the can I use service. This service provides information about the compatibility of HTML tags and attributes with different web browsers. It allows us to check whether a specific tag or attribute is supported by the browsers we are targeting. Important! We can use this service also for attributes and CSS properties, Javascript methods, browsers Api and everything related to web development. This service link is: Service: Can I Use

Almost useful web service is CANNCLUDE, who is checking can er some tag put in another tag. This service is very useful, because, it checks if we use some tag in another tag, which is not allowed. For example, we can't put div tag in span tag. This service link is: Service: CANNCLUDE

We need always to check the validation of us HTML code. Also, when we later will learn CSS, we need to check the validation of our CSS code. This service is provided by W3C Validator. It is important, because, if we don't do it, we can have problems with displaying our page in different browsers. And, we can lose some functionality of our page.

The CSS validator: W3C CSS Validator

In W3C Validator we can add URL, add with copy / paste code, or, we can upload us HTML file. It is very useful, because, we can check the validation of our page in different ways.

If we see yellow or red lines, it means that we have some problems with our code. If we see green lines, it means that our code is valid and we can use it without any problems. Yellow lines mean that we have some warnings, but, our code is valid. Red lines mean that we have some errors in our code, and, we need to fix them. If we see green lines, it means that our code is valid and we can use it without any problems.
