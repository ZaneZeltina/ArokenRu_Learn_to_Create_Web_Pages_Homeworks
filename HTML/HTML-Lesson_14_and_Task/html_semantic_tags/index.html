About Semantic in HTML

Semantic tags provide meaning to the web content. They help search engines and other user agents understand the structure and context of the content on a web page.

Look at the Semantic tags used in this page. They are used to define the header, footer, main content, and sections of the page. This helps both users and search engines to better understand the layout and purpose of the content.

Semantic HTML is important for accessibility, SEO, and overall web standards. It allows developers to create more meaningful and structured documents that can be easily interpreted by both humans and machines.

Examples of semantic tags include <header>, <footer>, <article>, and <section>, <nav>, <aside>.

If, we want to show some content that is not semantic, we can use the <div> tag. However, it is recommended to use semantic tags whenever possible to enhance the meaning and structure of the content.

We can use <article> to represent a self-contained piece of content that could be distributed and reused independently, such as a news article or blog post. We use it, if the content is independent and can stand alone, like a blog post or news article. It is not used for content that is part of a larger document or section, such as a comment or a part of a blog post.

But, tag <section> is used to define a section of content that is related to a specific topic or theme. It is typically used to group related content together, such as a group of articles or a set of related images.

Tag <aside> is used to define content that is related to the main content of the page, but is not essential to its understanding. It is typically used for sidebars, pull quotes, or other supplementary content that provides additional context or information.

Tags <header> and <footer> are used to define the header and footer of a page or section. The header typically contains the main heading, navigation links, page logo and other introductory content, while the footer contains copyright information, links to related content, and other closing information. Tag <header> may be used multiple times in a page, for example, if we have multiple sections or articles, each with its own header. However, it is not recommended to use it for content that is not related to the main content of the page, such as a sidebar or a footer.

Tag <nav> is used to define a navigation section of the page. It typically contains links to other pages or sections of the site, and is used to help users navigate the site. It is not used for content that is not related to navigation, such as a sidebar or a footer.

Title tags <h1> , <h2>, <h3>, <h4>, <h5>, <h6> are used to define headings and subheadings in a document. They help to create a hierarchy of content and improve the document's structure and readability.
- <h1> is used for the main heading of the page.
- <h2> is used for subheadings.
- <h3> is used for sub-subheadings, and so on.
- <h4> is used for sub-sub-subheadings, and so on.
- <h5> is used for sub-sub-sub-subheadings, and so on.
- <h6> is used for the smallest headings in the document.

Important! We need to use only one <h1> tag per page, as it represents the main heading of the document. Other headings should be used in a hierarchical manner, with <h2> for subheadings, <h3> for sub-subheadings, and so on. Other tags like <h2>, <h3>, <h4>, <h5>, <h6> should be used accordingly in us page.
