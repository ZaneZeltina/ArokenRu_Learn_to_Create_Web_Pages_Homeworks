Forms tags has a attribute: disabled and readonly. The disabled attribute prevents the user from interacting with the form control, while the readonly attribute allows the user to view the content but not modify it. These attributes can be useful for creating forms that require certain fields to be read-only or disabled based on specific conditions.

Why we need to use attribute disabled? If it is used, the data from the form will not be submitted to the server when the form is submitted. This can be useful for fields that are not relevant to the current user or situation, or for fields that are meant to be read-only.

When we use attribute readonly, the user can still interact with the field, but they cannot change its value. This can be useful for fields that display information that should not be modified, such as a user's name or email address.

What is a difference between disabled and readonly? The main difference is that a disabled field cannot be interacted with at all, while a read-only field can be viewed but not modified. This means that a disabled field will not be included in the form submission, while a read-only field will be included but cannot be changed by the user. If same tag has a disabled attribute, we cannot use focus on it, and it will not be submitted with the form. If same tag has a readonly attribute, we can focus on it, but we cannot change its value, and it will be submitted with the form.

Obligate attribute placeholder is used to provide a hint to the user about what information is expected in the field. It is displayed as a grayed-out text inside the input field until the user starts typing. This can help improve the user experience by providing guidance on what to enter in each field.

Attribute inputmode is used to specify the type of keyboard that should be displayed when the user interacts with the input field. This can be useful for fields that require specific types of input, such as numeric or email addresses. For example, if we set inputmode="numeric", the user will see a numeric keypad when they focus on the input field. It is easy to use, especially on mobile devices, but it is not a best practice. It is better to use tag <input type="number"> for numeric input, or tag <input type="email"> for email input. Attribute inputmode has several values, such as text, numeric, decimal, tel, email, and url. Each value corresponds to a specific type of input that the user is expected to enter.

However, a better practice is to use the appropriate <input> type for the field, such as type="text", type="number", or type="email". This ensures that the correct keyboard is displayed on mobile devices and provides better validation for the input. Also, if we use attribute type users cannot enter invalid data, such as letters in a numeric field or special characters in an email field. This helps to ensure that the data entered by the user is valid and can be processed correctly by the server. But if we use attribute inputmode, users can enter invalid data, such as letters in a numeric field or special characters in an email field. This can lead to errors and inconsistencies in the data submitted by the user.

Attribute autocomplete is used to specify whether the browser should automatically fill in the field with previously entered values. This can be useful for fields that are frequently used, such as email addresses or usernames. The autocomplete attribute can have several values, such as on, off, name, email, username, and more. Each value corresponds to a specific type of input that the user is expected to enter. It is easy to use, when we open some site with form many times and browser remembers our data, and we can use it again. But it is not a best practice. It is better to use attribute autocomplete with value off, to prevent the browser from automatically filling in the field with previously entered values. This ensures that the user has to enter the data manually, which can help to prevent errors and inconsistencies in the data submitted by the user.

Important! In attributes name and id we must use unique values for each form control. This is important for several reasons:

  • Accessibility: Unique values for name and id attributes help screen readers and other assistive technologies to identify and navigate through the form controls.
  • JavaScript: If we want to manipulate the form controls using JavaScript, unique values for name and id attributes are essential for selecting the correct element.
  • Form Submission: When the form is submitted, the server expects unique names for each form control to correctly process the data.

How we can these unique values? We can open some documentation about reserved words in HTML, and use them as unique values for name and id attributes. For example, we can use userName, userEmail, userAge, etc. This will help us to avoid conflicts with other form controls and ensure that the form works correctly.

We can use the attribute autofocus to automatically focus on a specific input field when the page loads. This can be useful for improving the user experience by allowing users to start typing without having to click on the input field first. When we are open some page who has interactive elements, such as forms, links or buttons, the browser will automatically focus on the first input field with the autofocus attribute. In Google Chrome we see blue 1px border around the interactive element.

We know about status: focus. It is used to all interactive elements in pages. We can navigate through the interactive elements on a page using the Tab key. When we press the Tab key, the browser will move the focus to the next interactive element on the page. If we want to skip some interactive elements, we can use the Shift + Tab key combination to move the focus to the previous interactive element. We can press the Enter key to activate the focused element, such as a button or a link. This allows users to interact with the page without using a mouse or touchpad, which can be especially useful for users with disabilities or those who prefer keyboard navigation.

Forms tag <label> has a for attribute, which is used to associate the label with a specific form control. This is important for accessibility, as it allows screen readers to correctly identify the label for the form control. When we click on the label, the browser will automatically focus on the associated form control, making it easier for users to interact with the form.

In tag <input we can use attribute form to specify the form to which the input element belongs. This is particularly useful for input elements that are not contained within a <form> element but still need to be submitted with a specific form. By using the form attribute, we can associate the input element with the desired form by referencing the form's id. We can connect the input element who isn't inside the <form tag. We use this attribute to get as possible to send data to the server.

Important! If, we use attribute form in tag <input>, we must use the id attribute in the <form> tag to specify the form to which the input element belongs. The value of the form attribute in the <input> tag should match the id of the <form> tag. This allows us to connect the input element to a form that is not directly contained within the <form> tag.

Important! Don't confuse tag <form> with the attribute form in tag <input>. The <form> tag is used to create a form, while the form attribute in the <input> tag is used to associate the input element with a specific form. The <form> tag is a container for all form controls, while the form attribute in the <input> tag allows us to connect an input element to a form that is not directly contained within the <form> tag.

Tag <input> has attributes minlength and maxlength, which are used to specify the minimum and maximum length of text that can be entered into the input field. This is useful for validating user input and ensuring that it meets certain criteria before being submitted.

Look for at specific attribute in tag <input>:size. The size attribute specifies the width of the input field in characters. This can be useful for controlling the appearance of the input field and ensuring that it is large enough to accommodate the expected input. However, it is not a best practice to use the size attribute, as it does not provide a consistent user experience across different devices and screen sizes. Instead, it is better to use CSS to control the width of the input field.

But in tag <select> attribute size is used to specify the number of visible options in a dropdown menu. This can be useful for creating a multi-select dropdown where users can select multiple options at once. The size attribute can have a value greater than 1, which will display the specified number of options in the dropdown menu without requiring the user to click on it.

Form tags <input> and <textarea> has a specific attribute pattern, which is used to specify a regular expression that the input value must match in order to be considered valid. This is useful for enforcing specific input formats, such as email addresses or phone numbers. In attribute pattern we can use regular expressions to define the expected format of the input value. For example, we can use the pattern ^[a-zA-Z0-9]+$ to allow only alphanumeric characters in the input field. If the input value does not match the specified pattern, the form will not be submitted, and an error message will be displayed to the user.

Also, we can use the oninvalid event handler to customize the validation feedback for users. This event is triggered when a form control fails to validate, allowing us to display a custom error message or take other actions to assist the user. In this attribute value we can use JavaScript code to handle the invalid input. For example, we can use the oninvalid event handler to display a custom error message when the user enters an invalid value in the input field. This can help to improve the user experience by providing clear feedback on what went wrong and how to fix it. We will learn more about this, when start to learn JavaScript.

Form tags <input> and <textarea> has a specific attribute spellcheck, which is used to enable or disable spell checking for the input field. This can be useful for ensuring that users enter text without spelling errors, especially in fields where accuracy is important, such as email addresses or names. The spellcheck attribute can have a value of true or false. If set to true, the browser will check the spelling of the input text and provide suggestions for corrections. If set to false, spell checking will be disabled for the input field. This attribute values can be used to control the behavior of spell checking in the input field. For example, we can use the spellcheck="true" attribute to enable spell checking for the input field, or spellcheck="false" to disable it.

Form has a attribute novalidate, which is used to disable the browser's default form validation. This can be useful when we want to implement custom validation logic using JavaScript or when we want to submit the form without validating the input fields. When the novalidate attribute is present, the browser will not perform any validation on the form controls before submitting the form. This means that even if required fields are empty or if the input values do not match the specified patterns, the form will still be submitted. However, it is important to note that disabling validation can lead to invalid data being submitted to the server, so it should be used with caution. We used this attribute, when the form validation is handled by JavaScript or when we want to submit the form without validating the input fields. This can be useful for forms that require complex validation logic or for forms that need to be submitted without any validation.

Form tag <textarea> has two attributes: rows and cols. The rows attribute specifies the number of visible text lines in the textarea, while the cols attribute specifies the width of the textarea in characters. These attributes can be used to control the size of the textarea and ensure that it is large enough to accommodate the expected input. But, the best practice is to use CSS to control the size of the textarea, rather than using the rows and cols attributes. This ensures that the textarea is responsive and adapts to different screen sizes and devices.

Textarea attribute wrap is used to specify how the text in the textarea should be wrapped. The wrap attribute can have two values: soft and hard. If set to soft, the text will wrap at the end of the line without inserting line breaks. If set to hard, the text will wrap at the end of the line and insert line breaks. This can be useful for controlling how the text is displayed in the textarea and ensuring that it is easy to read and edit. It works with the cols attribute. It ads line breaks to the text in the textarea based on the specified width. This can be useful for ensuring that the text is displayed correctly and is easy to read. However, it is not a best practice to use the wrap attribute, as it does not provide a consistent user experience across different devices and screen sizes. Instead, it is better to use CSS to control the wrapping of text in the textarea.
